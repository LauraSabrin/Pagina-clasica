¿Qué es Javascript?

Empecemos por el principio: en 1995, y en base a un navegador actualmente inexistente:NetScape , surge este lenguaje de programación a manos de vBrendan Eich, cuyo objetivo sería interactuar con las acciones del usuario, además de ganar versatilidad en diseño. En 1998. Microsoft crea el elemento XML HttpRequest: el "padre" de AJAX, que nos permite cargar o interactuar con contenido de backend si recargar la página al completo. En el año 2000 Douglas Crockford crea una forma de estandarizar objetos en Javascript: el documento JSON, "javaScript Object Notation": un formato de texto sencillo para el intercambio de datos, considerado un formato independiente del lenguaje debido a que cobrço importancia como alternativa a XML En el año 2004 Gmail empezó a utilizar a Javascript y AJAX para optimizar sus procesos, por lo que el auge del lenguaje era ya una realidad, siendo así que en 2006 el afamado programador y empresario John Resig crea la librería jQuery, software libre y de código abierto, diseñado con una sintaxis cuya meta es facilitar la navegación por un documento, crear animaciones, desarrollar aplicaciones AJAX entre otros.

Diferencias con Java:

  • JavaScript es un lenguaje interpretado: su código fuente se ejecuta directamente, instrucción a instrucción; no pasa por un proceso de compilación sino que un programa intérprete lee la instrucción a tiempo real y la ejecuta. Por el contratio, Java es un lenguaje compilado: es convertido directamente a código que el procesador puede ejecutar, así que es más rápido y eficiente que un lenguaje interpretado, además de ofrecer al desarrollador@ más control sobre aspectos del hardware.

  • Java está orientado a objetos puros mientras que JavaScript puede emular la prorgamación orientada a objetos
  • JavaScript es dinámico, flexible y sencillo a la hora de trabajar, pero Java tiene variables definidas que no se pueden modificar, lo que lo hace más complejo.

  • El tipado de Java es fuerte, más estricto en cuanto al tipo de dato que se puede asignar a una variable; esta definición no se puede cambiar, lo que da lugar a menos margen de error que con Javascript: este tiene un tipo de tipado más débil, donde una misma variable puede contener texto, luego un objeto, array o número.

¿Qué puedo hacer con Javascript?💻

Puede agregar nuevo HTML a la página, cambiar el contenido existente y sus estilos. Además, puede reaccionar a las acciones de los usuarios, recordar datos en el lado del cliente con el almacenamiento local, obtener y configurar cookies, hacer preguntas a los visitantes y mostrar mensajes. También, puede enviar solicitudes a servidores remotos, carga y descargar archivos.

  • Ventajas:

    1. Es multiplataforma:

      2. el intérprete suele estar en varios sistemas operativos, por lo que no hay que adaptar el código a una plataforma en concreto.

    2. Portabilidad:

      3. el mismo programa puede llevarse a diferentes plataformas

    3. Mayor rendimiento

      4. Al ser lenguaje interpretado se ejecuta en el navegador cliente, lo que disminuye la carga de trabajo del servidor web

      Es decir: integra completamente HTML y CSS, simplifica y es soportado por la mayoría de navegadores (habilitado de forma predeterminada).

Desventajas

  1. Seguridad del lado del cliente:

    2. como su código se ejecuta en el navegador del usuario, hay riesgo de ataques como inyección de código o cross-site-scripting, por lo que hay que tomar medidas para asegurar el código y prevenir estos ataques.

  2. Bajo rendimiento en aplicaciones complejas:

    3. al se un lenguaje interpretado, su rendimiento puede verse afectado a pesar de que los navegadores actuales hayan mejorado su ejecución; hablamos en casos de webs muy complejas o dispositivos con recursos limitados. Su tipo de lenguaje es, como véis, tanto una posible ventaja como un posible obstáculo en el camino.

  3. Dificultad para indexar en motores de búsqueda:

    4. el contenido dinámico generado por este lenguaje complica la indexación para muchos motores de búsqueda aún.

  4. Mantenimiento y legibilidad complicados:

    5. conforme las aplicaciones web se vuelven más o grandes o complejas, este lenguaje de programación puede llegar a ser difícil de entender si uno no tiene muy dominado y organizado su código
